# UI/UX Components (The "Look")

## 4.1. Design Philosophy
*   **Clean & Focused:** Content-first approach. The UI gets out of the way of the conversation.
*   **Responsive & Fluid:** CSS Grid layout with CSS variables for precise, calculated positioning (e.g., `--layout-sidebar-screen-position`).
*   **Themable:** Built on DaisyUI, supporting 20+ themes out of the box with a custom default theme defined in `app.css`.

## 4.2. Core Components
*   **MessageComposer:** Fixed at the bottom, auto-resizing textarea, disabled state during generation.
*   **ChatHistory/Folder:** Hover states reveal action buttons (Rename, Delete, Color). Uses CSS `@utility` classes for DRY styling.
*   **ToastContainer:** Global, z-indexed notification system with animated entry/exit (Svelte `fly` transitions).
*   **ErrorBoundary:** Catches component crashes, displays a fallback UI, and prevents the whole SPA from white-screening.

## 4.3. Interaction Patterns
*   **Optimistic Updates:** Renaming a chat updates the sidebar instantly; errors revert the state and show a toast.
*   **Streaming Indicators:** A loading spinner (`loading-dots`) appears inline with the assistant's message bubble while the stream is active.
*   **Hover Intent:** Action buttons (edit/delete) on sidebar items only appear on hover to reduce visual clutter.

## 4.4. Responsive Design Rules
*   Sidebar collapses completely via the `hideSideMenu` store.
*   Main layout uses CSS Grid `[grid-area]` for robust reflowing.
*   Max-width constraints (`--layout-app-max-width: 72rem`) ensure readability on ultra-wide monitors.
